Formas de declarar variables

  • Usando var delante
  • Usando let delante
  • Usando const delante
  • Usando nada

Por buenas prácticas, usaremos siempre una de las 3 primeras.

Var

var es la forma clásica de declarar una variable desde los inicios de JavaScript

let y const fueron añadidos al lenguaje en 2015

var se usa principalmente como método de compatibilidad con navegadores antiguos.

const y let

Por regla general las variables se declaran con const.

Sin embargo el valor de las variables puede canbiar usaremos let

EJemplos con variables fijas o constantes y de valor variable. Ref al JS linea 1.

El operador de asignación =

Este operador en JS, el signo de igual = no equivale a "resultado de" como en algebra. Si no que es un operador de asiganación.

x = x + 5

Esta expresión matemáticamente no tiene sentido pues nunca va a haber una número que equivalga a una suma de si mismo con 5.

Sin embargo, en código, no solo tiene mucho sentido si que es una práctica muy común . Esta expresión indica que el actual valor de x le estamos asignando el valor de la suma de x + 5.

BAsicamente, hemos aumentado el valor de x en 5.

Nota: En JavaScript el operador "igual a" se escribe ==

Tipos de datos

Las variables pueden tener distintos tipos de datos. Pueden ser número como 100 o texto como "Juan Eduardo".

En programación, a los valores de texto se les llama cadenas de caracteres o strings

Los strings se escriben dentro de comillas dobles o simples.

Si escribes un número dentro de unas comillas , será tratado como un string de texto . Ref al Js linea 1.

Value = Undefined

A menudo en los programas informáticos se declaran sin un valor. El valor puede que sea algo que deba ser calculado o que sea provisto por el usuario, como un input.

Las variables declaradas sin valor tendrán como valor Undefined. Ref al JS línea 25.

Declarar varaiables

Si se redeclara una variable usando var, la variable mantedrá su valor.

Sin embargo, al hacear esto con una variable declarada con let o const el código lanzará un error.

Variables con let deben ser declaradas antes de usarse (hoisting)

Usando var podemos usar variables mientras estas esten declaradas en cualquier parte del código

con let la declaración debe ser anterior al uso u obtendremos un error de referencia. Red al Js línea 29
